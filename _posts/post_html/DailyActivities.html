About the Data

This exploratory analysis will examine the amount of time T. L. Bailey (2019) spent completing different kinds of activities over a period of 28 days. Each day, Bailey recorded the amount of time they spent completing each of the following activity types: Screen time, social networking, reading/reference, productivity, health/fitness, entertainment, creativity, and other. Bailey also indicated whether they practiced yoga each day.

Prior to examining the data, it is clear that the conclusions that can be drawn from this data are limited due to the following limitations of the data set:

  1. Data is recorded for only 1 person. This small sample size means it isnt appropriate to conduct hypothesis testing.
  2. Data is recorded for a short period of time, meaning it isnt possible to meaningfully assess trends across time.
  3. The available documentation for the data does not specify the unit of time (minutes, hours, etc.). For the purposes of this analysis, I assume the unit is minutes.
  4. The activity types are not clearly defined and may not be mutually exclusive. For example, screen time may include social networking, entertainment, etc.

Data Source: Bailey, T.L. (2019). Screen Time Data. https://data.world/taylynners04

The data frame is pretty clean, but some things require attention:

  • Column names are delimited by periods where the original csv file contained spaces. For personal preference, Ill replace all instances of . with _ .

  • Date contains character data. Ill convert it to type Date and check for anomalies. This will allow me to extract specific pieces of date-related information from this variable, which will come in handy below

  • Week.Day contains character data indicating the day of the week. I could assume that the week day recorded for each date is correct, but instead Im going to extract the calendar-observed week day associated with each date, which Ill have access to once the Date variable is in Date format, and use those values as Week.Day. In doing so, Ill convert this variable to type factor with one level for each day of the week.

  • Total.Screen.Time to Creativity contain integer data representing time spent on each activity. As previously mentioned, the unit of time is not specified but I assume that it is minutes. Ill check these variables for anomalies which may suggest a different unit.

  • Total.Screen.Time follows a naming scheme different from the other variables, so Ill change the name to Screen .

  • Yoga contains integer data and indicates whether yoga was practiced that day. Because this binary variable represents two levels of a condition, Ill convert Yoga to type factor. Based on the documentation and data, it is apparent that 0 = No (yoga not practiced) and 1 = Yes (yoga practiced). Ill also check for anomalies, such as the presence of an unexpected third level.

Im going to start by checking the skewness and kurtosis of each variable to get an initial sense of the distributional properties. Even though Im not going to use inferential statistics to analyze the data, I still want to know whether the data are approximately normally distributed because this informs whether its more appropriate to use the mean or median to describe the data.

The last three variables (Health_and_Fitness, Entertainment, and Creativity) have a very strong positive skew and a very steep peak, indicated by skewness and kurtosis greater than 2, which suggests that these variables dont follow a normal distribution. Reading_and_Reference also looks like it probably suffers from non-normality. The remaining variables are also a bit positively skewed and a bit misshapen, but less egregiously so.

Unsurprisingly, theres a bit of mismatch between the mean and median of each variable - more so for the variables that clearly follow a non-normal distribution. Based on all this, Im going to describe the data using the median moving forward.

Now that thats decided, Im going to take a look at the median for each variable. Bailey spent more time using screens (Mdn = 111 minutes/day) than they spent on any of the other activity types included in the data set. They spent the second most time social networking (Mdn = 58 minutes/day). They either spent very little time on all other activity types or they did not record data as diligently for them (Mdns between 0 and 9).

Data Visualization

Histograms

To understand the data a bit better, Im going to create histograms. Looking at each variables distribution will cement the conclusions I made above about normality (or lack thereof), but more importantly its a first step in identifying whether anything interesting might be going on.

Im going to write a function named plot_hist that constructs a histogram for each of the 8 activity variables. The function, named plot_hist, will take two arguments: colname, which is a vector of column names for the variables that Im plotting, and df, which is the data frame.

Based on the descriptive statistics for each variable, its clear that each distribution will be vastly different. This means that different plots will require different parameters, so the range of data (xrange), x-axis tick increments (xinc), maximum x-axis tick (xmax), bin width (xbin), and bin center (center) are conditional on properties of the data for each variable.

The plots are made in a multi-step fashion. First, an initial plot is made using the conditional values I just mentioned. Second, I access the existing plots parameters, extract the maximum y-value, and use this to determine y-axis tick increments (yinc) and the maximum y-axis tick (ymax). This must be done after the initial plot is made because you cant access parameters for a plot that doesnt exist yet. Finally, I modify the original plot by adding the new, conditional y-axis parameters.

plot_hist <- function(colname, df) {
  
  # Compute range of x axis data as distance of maximum value from 0
  xrange <- max(df[ , colname]) - 0
  
  # If range of x data is high, set x ticks to increment by 20
   if (xrange >= 100) {
    xinc <- 20
  }
  
  # If range of x data is moderate, set x ticks to increment by 10
  else if ((xrange < 100) & (xrange >= 40)) {
    xinc <- 10
  }
  
  # If range of x data is small, set x ticks to increment by 2
  else if (xrange < 40) {
    xinc <- 2
  }
  
  xmin <- 0
  
  # Round up to the nearest multiple of xinc and set as maximum x tick
  xmax <- ceiling(max(df[ , colname]) / xinc) * xinc
  
  # Set the bin width so there are two bins per tic
  xbin <- xinc / 2
  
  # Center bar on middle of the bin
  center <- xbin / 2
  
  textsize <- 10
  
  # Create initial histogram with specified x-axis parameters
  plot <- ggplot(df, aes(x = .data[[colname]])) +
    geom_histogram(binwidth = xbin,
                   center = center,
                   fill = "deepskyblue4",
                   color = "black",
                   linewidth = .2) +
    scale_x_continuous(name = str_replace_all(colname, 
                                              c("_and_" = "/",
                                                "_" = " ")),
                       limits = c(xmin, xmax), 
                       breaks = seq(xmin, xmax, xinc)) +
    theme_minimal() +
    theme(axis.text = element_text(size = textsize),
          axis.title.x = element_text(size = textsize),
          axis.title.y = element_blank())
  
  ymin <- 0
  
  # Round the maximum y value to the nearest multiple of 5 and set as maximum y tick
  ymax <- ceiling(max(ggplot_build(plot)$layout$panel_scales_y[[1]]$range$range) / 5) * 5
  
  # If maximum y tick is high, set y ticks to increment by 5
  if (ymax > 5) {
    yinc <- 5
  }
  
  # If maximum y tick is high, set y ticks to increment by 1
  else if (ymax <= 5) {
    yinc <- 1
  }
  
  # Modify initial plot with additional y axis parameters
  plot <- plot + scale_y_continuous(limits = c(ymin, ymax),
                                    breaks = seq(ymin, ymax, yinc))

}

My function takes a vector of column names for the colnames argument. I could use the colnames function to extract the names from the data frame directly, but instead Im creating a vector manually so the variables are in the order Ill ultimately want them to be in. I selected this particular order so plots with more similar axes can be more easily positioned next to each other.

activities_to_plot <- c("Screen", "Social_Networking",

Now, Im going to call the plot_hist function. Im doing this in lapply so the function will be called for each variable in my vector of column names, with parameters will be computed for each variable from the activ_data_clean data frame, and each plot is added to a list named hist_list.

hist_list <- lapply(activities_to_plot, plot_hist, df = activ_data_clean)
